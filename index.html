Quickstart

Compose CSS is on GitHub.

Download the compose CSS file and put it into the css folder of your website project.

Reference the CSS file in your HTML:

<link rel="stylesheet" type="text/css" href="/css/compose.css">

By then you are ready to use Compose CSS.

There is also a base compose CSS available â€“ with only half the size.

It can be used as a faster-loading alternative with a reduced feature set. Compared to the full version, it has the following limitations:

  • Lightweight color system, it has only color edge values for color, background and border;
  • No form styling;
  • No max-width, min-width, max-height, and min-height classes;
  • No responsive breakpoints for font size, line height, height, max-height, min-height, width, max-width, and min-width;
  • No square classes.

Responsive Breakpoints

Heading size, font size, and line height can have different sizes based on minimum screen width. Address the different minimum screen sizes with responsive breakpoints, wich are indicated by CSS class prefix values of:

.sm-
The size is applied only if the screen is at least of small width.
.rg-
The size is applied only if the screen is at least of regular size.
.md-
The size is applied only if the screen is at least of mid size.
.lg-
The size is applied only if the screen is at least of large size.

E.g. .lg-fs-1, .md-h1, or .rg-lh-3.

A size class without a prefix will be used in case the alternate minimum screen size is not defined.

In addition there are the following color classes available:

.base-color (which is default text-color).

The color can be used as text color, background color and border color. It is defined through:

:root { --base-color = var(--neutral-color-d4); }
.base-background

The color can be used as text color, background color and border color. It is defined through:

:root { --base-background = white; }
.black

The color can be used as text color, background color and border color. It is defined through:

:root { --black = black; }
.white

The color can be used as text color, background color and border color. It is defined through:

:root { --white = white; }
.mark, mark

The color is defined through:

:root { --mark-background: yellow; }
.kbd, kbd

The color is defined through:

:root { --kbd-background: var(--neutral-4); }

Images

Responsive Images

Images are by default responsive and take the entire available horizontal space (up to the width of the image) while preserving aspect-ratio.

img/ .width-100 //will consume the entire available width, may enlarge the image while preserving aspect ratio

Contained Images

If the image has a fixed width or fixed height, the usage of .fix-contain will put the entire visible image inside of the given dimensions while preserving aspect-ratio and without stretching the image.

Combine .fit-contain with fit positions

  • .fit-t (top),
  • .fit-tl (top-left),
  • .fit-tr (top-right),
  • .fit-r (right),
  • .fit-b (bottom),
  • .fit-bl (bottom-left),
  • .fit-br (bottom-right), and
  • .fit-l (left).
img/ .fit-contain
img/ .fit-contain.fit-tl

Cover Images

If the image has a fixed widht or fixed height, the usage of .fix-cover will consume the entire available space of the given dimensions while preserving aspect-ratio.

The image might be stretched to cover.

img/ .fit-cover

Background Images

Background images can also be positioned with .fit-cover, which is useful for hero banners.

Responsive Breakpoints

In addition .width, .max-width, and .min-with classes can be altered with responsive breakpoints, wich are indicated by CSS class prefix values of:

.sm-
The width is applied only if the screen is at least of small width.
.rg-
The width is applied only if the screen is at least of regular width.
.md-
The width is applied only if the screen is at least of mid width.
.lg-
The width is applied only if the screen is at least of large width.
